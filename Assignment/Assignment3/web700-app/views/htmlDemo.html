HTML5 Semantic Elements

Overview of HTML5 Semantic Elements
Element Description Example Use
<header> Defines a header for a document or a section Site logo, navigation links, search form
<nav> Defines navigation links Primary site navigation
<section> Defines a section in a document Chapters, news items, contact information
<article> Defines an independent, self-contained content Forum posts, magazine and newspaper articles
<aside> Defines content aside from the content it is placed in Sidebars, call-out boxes
<footer> Defines a footer for a document or a section Authorship information, contact information, sitemap
Total Elements Covered 6

This table provides an overview of various HTML5 semantic elements, showcasing their importance in structuring web content semantically. The use of these elements helps in creating more accessible and SEO-friendly web pages.

HTML5 Features

  1. HTML5 Doctype: Simplifies the document type declaration and ensures compatibility across browsers.
  2. Semantic Elements: Provides meaning to web content with tags like <article>, <section>, and <nav>.
  3. Form Enhancements: Introduces new form controls like <input type="email"> and <input type="date">.
  4. Video and Audio: Enables multimedia integration without third-party plugins using <video> and <audio> elements.
  5. Graphics and 3D Effects:
    • <canvas>: Allows dynamic rendering of 2D shapes and bitmap images.
    • <svg>: Supports inline SVG for scalable vector graphics.
    • WebGL: Facilitates rendering interactive 3D graphics within any compatible web browser.
  6. Geolocation API: Enables users to share their location with trusted web applications.
